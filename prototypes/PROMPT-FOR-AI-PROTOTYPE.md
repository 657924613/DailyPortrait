# DailyPortrait 全页面 UI 原型设计提示词

## 你的角色

你是一位高级移动端 UI/UX 设计师，擅长用 HTML/CSS/JS 制作高保真交互原型。你需要为一款 Android App 设计完整的界面原型。

---

## 产品概述

**产品名称**：每日肖像（DailyPortrait）

**一句话描述**：每天拍一张自拍，一年后自动合成延时视频，看见时间在脸上留下的痕迹。

**核心用户**：18-35 岁，想记录自己变化的普通人。不是摄影爱好者，不追求完美照片，追求的是"坚持"和"回看时的惊喜"。

**情感关键词**：仪式感、私密档案、时间的力量、温暖克制、不炫耀

**竞品参考**（视觉气质，非功能）：Day One 日记、Linear 的克制感、胶片相机 App 的质感

---

## 功能页面清单（共 7 个页面）

### 1. Onboarding 引导页（3 页滑动）

**目的**：让新用户 3 秒内理解产品价值，产生"我想试试"的冲动。

**内容**：
- 第 1 页：核心价值 —— 每天一张，记录真实的自己
- 第 2 页：核心功能 —— 洋葱皮叠加帮你对齐构图
- 第 3 页：核心回报 —— 365 天变成延时短片

**交互要求**：
- 支持左右滑动切换
- 有页面指示器
- 右上角"跳过"按钮
- 最后一页按钮文案变为"开始记录"
- **点击"开始记录"时需要一个有冲击力的转场动画**（收缩/展开/翻转/光效均可，要有"新世界打开"的感觉）

**设计禁忌**：
- 不要用 emoji 当图标
- 不要纯文字 + 大片空白
- 不要像功能说明书

### 2. Dashboard 主页（有数据状态）

**布局元素**：
- 顶部：问候语（今天早上/下午/晚上）+ 第 N 天
- 操作入口：拍照按钮、设置按钮、导出视频按钮
- 今日打卡卡片：大图区域（已拍/未拍两种状态）+ 日期 + 状态文案
- 数据卡片：连续天数、本周进度（7 天圆点）
- 历史相册网格：3 列缩略图，底部有日期标注
- "统计"入口链接

**交互**：
- 点击今日卡片 → 进入相机
- 长按历史照片 → 删除
- 点击历史照片 → 大图浏览

### 3. Dashboard 空状态（首次打开，无照片）

**布局**：
- 大标题："记录从今天开始"
- 副标题：鼓励性文案
- 一个醒目的 CTA 按钮："拍下第一张"
- 设置入口

### 4. 相机页

**布局元素**：
- 全屏取景框
- 洋葱皮叠加层（上一张照片半透明覆盖，帮助对齐）
- 人脸对齐引导圆环（居中）
- 底部：快门按钮（大圆）+ 关闭按钮
- 拍摄后：预览 + "使用这张" / "重拍" 两个按钮

### 5. 设置页

**内容**：
- 每日提醒开关 + 时间选择
- 关于（版本号、重看引导入口）
- 隐私政策 / 用户协议链接
- 调试工具（仅开发模式显示）

### 6. 视频导出设置面板（底部弹出 Sheet）

**内容**：
- 帧率滑块（2-20 fps）
- 画质选择（流畅 / 标清 / 高清 / 超清）
- 仅使用对齐照片开关
- "生成视频"按钮
- 生成中：进度条 + 预计剩余时间
- 生成完成："立即查看"按钮

### 7. 统计页

**内容**：
- 三个数据卡片：总天数 / 当前连续 / 最长连续
- 月度热力图日历（拍过的天高亮，未拍的淡色）
- 月份前后切换

## 设计约束

### 配色要求
- **禁止使用黑暗/深色主题**
- 背景应为暖色调浅色（米色/象牙白/暖灰/纸张色均可）
- 需要一个强调色用于 CTA 和重点标注（不限定具体颜色，但要克制使用）
- 照片区域应该是页面中唯一的"彩色来源"，其余 UI 元素保持中性
- 整体色彩不超过 3 种有色（背景 + 文字 + 强调色），其余靠灰阶层级

### 字体与排版
- 中文无衬线体（系统默认即可）
- 标题用极细或极粗字重制造对比（不要全部 regular）
- 数字可以用大号 + 细体来制造"editorial"感
- **全部文案必须是中文，禁止出现英文**

### 视觉风格
- 圆角统一（建议 16-28px 之间选一个值）
- 阴影极淡或不用阴影（用边框/背景色差区分层级）
- 不要渐变色块装饰
- 不要 emoji 作为图标（用简洁的线性图标或抽象图形）
- 留白要充足，不要信息密度过高

### 交互与动效
- 页面切换要有过渡动画（不要硬切）
- 按钮要有按下反馈（scale 或 opacity 变化）
- Onboarding 到 Dashboard 的转场是重点，要有"仪式感"
- 列表/卡片入场可以有轻微的 stagger 动画

### 尺寸
- 按 390×844 (iPhone 14 / Pixel 7 尺寸) 设计
- 触控目标最小 44×44pt
- 底部留出安全区域（home indicator）

---

## 输出要求

1. 输出为**单个 HTML 文件**，包含所有页面和交互
2. 页面之间可以通过按钮/手势切换（模拟真实 App 导航）
3. 用 CSS 变量管理配色，方便后续调整
4. 代码结构清晰，每个页面用注释分隔
5. 在文件顶部用注释说明你的设计思路（为什么选这个配色/布局/动效）

---

## 加分项（如果你能做到）

- Onboarding 的"开始记录"转场动画有创意且流畅
- Dashboard 的照片网格有真实感（用渐变色块模拟不同照片）
- 相机页的洋葱皮叠加有视觉表现
- 统计页的热力图日历可以点击切换月份
- 整体有统一的设计语言，看起来像一个完整的产品而非拼凑

---

## 我不想看到的

- 像素级复刻 Material Design 默认样式
- 大面积纯白 + 蓝色强调色（太像 Google 默认）
- emoji 当图标
- 英文文案
- 功能说明书式的 Onboarding（"功能1：xxx"）
- 没有动效的硬切换
- 信息过载的 Dashboard（一屏塞太多东西）
